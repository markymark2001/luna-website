# Public Website Docs

This directory contains durable operating context for the static public website.

## Workflows

- `workflows/brainstorming.md`: use when shaping a new idea into a design or
  spec before implementation.
- `workflows/writing-plans.md`: use for multi-step implementation plans saved
  under `docs/plans`.
- `workflows/systematic-debugging.md`: use before fixing bugs, visual
  regressions, broken links, browser errors, or other unexpected behavior.
- `workflows/reviewing.md`: use when the sole task is codebase, branch, PR, or
  implementation review.
- `workflows/runtime-evidence.md`: use for local browser verification with the
  Codex browser.
- `workflows/specs.md`: use for product or durable design specs saved under
  `docs/specs`.

Supporting debugging techniques live under `workflows/debugging`.
